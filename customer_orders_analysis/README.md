This project demonstrates SQL reasoning and problem-solving skills by analyzing customer and order data. The focus is on translating business questions into SQL queries using joins, aggregation, and filtering logic.


Schema

customers(customer_id, name, signup_date)
orders(order_id, customer_id, order_date, status, amount)

Key Skills Practiced

LEFT vs INNER JOIN
Aggregations (COUNT, SUM, AVG)
GROUP BY and HAVING
Identifying customers with missing or qualifying data
Translating real-world questions into SQL logic

Each query includes comments explaining the reasoning behind the approach.
